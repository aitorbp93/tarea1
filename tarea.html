Análisis de Requisitos

Requisitos Funcionales y No Funcionales

Requisitos Funcionales

  • Catálogo de Productos
    • Función Principal: Los usuarios podrán explorar todos los productos organizados en diferentes categorías.
    • Subfunción: Al hacer clic en un producto, los usuarios podrán ver detalles como la descripción, el precio y características adicionales.
  • Carrito de Compras
    • Función Principal: Los usuarios podrán agregar productos a un carrito de compras.
    • Subfunción: El carrito permitirá actualizar la cantidad de productos y eliminar artículos.
  • Registro e Inicio de Sesión
    • Función Principal: Los usuarios podrán registrarse con su correo electrónico y gestionar su cuenta.
    • Subfunción: Los usuarios podrán iniciar sesión, revisar sus compras anteriores y actualizar información personal como la dirección de envío.
  • Historial de Compras
    • Función Principal: Los usuarios tendrán acceso a un historial de pedidos anteriores, donde podrán ver detalles de cada compra.
  • Proceso de Pago
    • Función Principal: Los usuarios podrán pagar utilizando varios métodos, como tarjeta de crédito o PayPal.
    • Subfunción: Verificar el costo del envío antes del pago y confirmar el pedido.
  • Notificaciones de Pedidos
    • Función Principal: Los usuarios recibirán notificaciones por correo electrónico sobre el estado de su pedido, como confirmación de compra y envío.
  • Seguimiento de Pedidos
    • Función Principal: Los usuarios podrán seguir el estado de sus pedidos en tiempo real una vez que hayan sido enviados.

Requisitos No Funcionales

  • Usabilidad
    • La interfaz debe ser fácil de usar e intuitiva, con una navegación fluida entre las secciones del catálogo, carrito de compras y gestión de cuentas.
  • Rendimiento
    • La aplicación debe soportar múltiples usuarios simultáneamente, manteniendo tiempos de respuesta rápidos, especialmente durante el proceso de pago.
  • Seguridad
    • El sistema debe manejar de manera segura los datos de los usuarios y sus transacciones, asegurando la privacidad de la información personal y los datos financieros.
  • Disponibilidad
    • La plataforma debe estar disponible 24/7, con un mínimo de tiempo de inactividad durante actualizaciones o mantenimiento.
  • Escalabilidad
    • El sistema debe ser capaz de manejar el crecimiento en la cantidad de productos, usuarios y transacciones sin degradar el rendimiento.
  • Compatibilidad
    • La aplicación debe ser accesible en distintos dispositivos (móviles, tablets, desktops) y navegadores modernos.
  • Mantenibilidad
    • El código de la aplicación debe estar bien documentado para facilitar futuras actualizaciones o correcciones.

Diseño de Arquitectura

Arquitectura del Sistema

Frontend

  • Tecnología: Desarrollado con React para una interfaz de usuario rápida y adaptable, optimizada para dispositivos móviles y tablets.

Backend

  • Tecnología: Implementado con Node.js y una API REST para gestionar las operaciones de usuarios, carrito de compras y pagos.
  • Base de Datos: Utiliza MongoDB para almacenar información de productos, usuarios y pedidos.
  • Pasarelas de Pago: Integración con Stripe para pagos con tarjeta y Coinbase Commerce para pagos con criptomonedas.

Componentes del Sistema

  • Interfaz de Usuario (Frontend)
    • React: Proporciona una experiencia de usuario ágil y responsiva.
  • API REST (Backend)
    • Node.js: Maneja las solicitudes y respuestas entre el frontend y la base de datos.
    • Operaciones CRUD: Permite crear, leer, actualizar y eliminar datos de productos, usuarios y pedidos.
  • Gestión de Pagos
    • Stripe: Procesa pagos con tarjeta de crédito.
    • Coinbase Commerce: Facilita pagos con criptomonedas.

Flujo de Datos

  • Interacción del Usuario
    • Frontend (React): Los usuarios interactúan con la interfaz para navegar por productos, gestionar el carrito y realizar pagos.
  • Comunicación con el Backend
    • API REST (Node.js): El frontend envía solicitudes al backend para obtener o modificar datos.
    • Base de Datos (MongoDB): El backend consulta o actualiza la base de datos según las solicitudes del usuario.
  • Procesamiento de Pagos
    • Stripe y Coinbase Commerce: El backend se comunica con las pasarelas de pago para procesar transacciones y confirmar pagos.

Implementación

Implementación del Sistema

  • Configuración de la API REST
    • Objetivo: Gestionar el catálogo de productos, el carrito de compras y los pagos.
    • Tecnología: Utilizar Node.js para crear una API REST eficiente y escalable.
  • Desarrollo del Frontend
    • Objetivo: Crear interfaces de usuario atractivas y responsivas.
    • Tecnología: Utilizar React para desarrollar una interfaz de usuario dinámica, incluyendo un sistema de recomendación de productos.
  • Integración de Opciones de Pago
    • Objetivo: Permitir a los usuarios realizar pagos de manera segura y eficiente.
    • Tecnología: Integrar Stripe para pagos con tarjeta y Coinbase Commerce para pagos con criptomonedas.
    • Funcionalidad Adicional: Configurar cupones de descuento en tiempo real para mejorar la experiencia del usuario.
